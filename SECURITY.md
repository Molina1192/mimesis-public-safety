# Security Policy

## Supported Versions

The Mimesis Public Safety project is under active development.

Security updates and fixes will be applied to the current development version.

| Version | Supported |
|---------|:---------:|
| Current Development | ✅ |
| Previous Releases | ❌ |

---

## Reporting a Security Issue

If you discover a security vulnerability affecting this repository or any associated tools, please do not create a public GitHub issue.

Instead:

1. Contact the project maintainer privately through GitHub.
2. Provide a detailed description of the issue.
3. Include reproduction steps, if applicable.
4. Include screenshots or logs when helpful.

The project maintainer will acknowledge receipt of the report as soon as reasonably possible and investigate the issue.

---

## Scope

Examples of security-related issues include:

- Accidental exposure of secrets or credentials.
- Vulnerabilities in scripts or utilities contained within the repository.
- Unsafe GitHub Actions or workflow configurations.
- Dependency vulnerabilities (if applicable).
- Unauthorized modification of production assets.

General bugs, feature requests, and design feedback should be submitted through the project's GitHub Issues page.

---

## Responsible Disclosure

Please allow the project maintainer reasonable time to investigate and resolve reported vulnerabilities before publicly disclosing them.

Responsible disclosure helps protect contributors and users while fixes are prepared.

---

## Security Best Practices

Contributors should:

- Never commit passwords, API keys, or authentication tokens.
- Verify files before committing them.
- Review third-party assets and tools before adding them to the repository.
- Keep local development environments secure and up to date.
- Follow the project's licensing and contribution policies.

---

## Questions

If you are unsure whether an issue is security-related, contact the project maintainer before creating a public issue.

Thank you for helping keep the Mimesis Public Safety project secure.

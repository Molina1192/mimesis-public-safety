# Security Policy

This document describes how security issues should be reported and handled for the **Mimesis Public Safety Project**.

The goal of this policy is to protect contributors, Project Assets, repository integrity, and any supporting tools used during development.

---

## Supported Versions

The Mimesis Public Safety Project is under active development.

Security fixes will be applied to the current development version.

| Version | Supported |
|---------|:---------:|
| Current Development | ✅ |
| Previous Releases | ❌ |

---

## Reporting a Security Issue

If you discover a security vulnerability affecting this repository or any associated tools, **do not open a public GitHub issue**.

Instead:

1. Contact the Project Owner privately through GitHub.
2. Provide a detailed description of the issue.
3. Include steps to reproduce the issue, when applicable.
4. Include screenshots, logs, or other supporting information if available.

Reports will be acknowledged as soon as reasonably possible and investigated in a timely manner.

---

## Scope

Examples of security-related issues include:

- Accidental exposure of secrets, credentials, or authentication tokens.
- Vulnerabilities in repository scripts or development utilities.
- Unsafe GitHub Actions or workflow configurations.
- Dependency vulnerabilities.
- Unauthorized modification of Project Assets.
- Malicious code or files introduced into the repository.
- Supply-chain risks involving third-party tools or assets.

General bugs, feature requests, design feedback, and documentation improvements should be submitted through the project's GitHub Issues page.

---

## Responsible Disclosure

Please allow the Project Owner reasonable time to investigate and resolve reported vulnerabilities before publicly disclosing them.

Responsible disclosure helps protect contributors, users, and the integrity of the project while corrective actions are prepared.

---

## Security Best Practices

Contributors are expected to:

- Never commit passwords, API keys, authentication tokens, or other secrets.
- Review files before committing them.
- Verify the licensing and integrity of third-party assets before adding them to the repository.
- Keep local development environments secure and up to date.
- Follow the project's contribution and licensing requirements.
- Report suspected security issues promptly.
